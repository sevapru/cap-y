## OpenCV 4.13 CUDA Build Flags

### CUDA Acceleration

| Flag | Value | Effect |
|------|-------|--------|
| `-DWITH_CUDA=ON` | ON | Enables all CUDA-accelerated OpenCV modules (DNN, image processing, feature detection) |
| `-DCUDA_ARCH_BIN=${CUDA_ARCH_BIN}` | `11.0` (Thor) | Compiles CUDA kernels for your specific GPU. Produces smallest, fastest binary. No JIT compilation at runtime |
| `-DCUDA_ARCH_PTX=` | empty | No PTX intermediate code. Prevents JIT compilation overhead on first run. Trade-off: binary only works on the specified architecture |
| `-DCUDA_FAST_MATH=ON` | ON | Enables `__fmul_rn`, fused multiply-add, and approximate math in CUDA kernels. Significant speedup for DNN inference. Tiny precision trade-off (< 1 ULP) acceptable for vision tasks |
| `-DWITH_CUBLAS=ON` | ON | Links cuBLAS for GPU-accelerated matrix operations. Benefits: faster `cv2.cuda.gemm()`, DNN layer computations, homography estimation |
| `-DWITH_CUDNN=ON` | ON | Links cuDNN for DNN inference. Required for GPU-accelerated `cv2.dnn` module |
| `-DOPENCV_DNN_CUDA=ON` | ON | Enables the CUDA backend for OpenCV's DNN module. Without this, `cv2.dnn.readNet()` falls back to CPU even with CUDA available |

**How to use in code:**
```python
import cv2

net = cv2.dnn.readNet("model.onnx")
net.setPreferableBackend(cv2.dnn.DNN_BACKEND_CUDA)
net.setPreferableTarget(cv2.dnn.DNN_TARGET_CUDA_FP16)  # FP16 for max throughput

gpu_mat = cv2.cuda_GpuMat()
gpu_mat.upload(frame)
resized = cv2.cuda.resize(gpu_mat, (640, 480))  # GPU resize
blurred = cv2.cuda.createGaussianFilter(...)     # GPU blur
```

### CPU Optimisation

| Flag | Value | Effect |
|------|-------|--------|
| `-DWITH_EIGEN=ON` | ON | Eigen3 for optimized linear algebra. Faster homography, fundamental matrix, PnP solving |
| `-DWITH_TBB=ON` | ON | Intel TBB thread pool for parallel CPU operations. Benefits multi-threaded `cv2.filter2D`, `cv2.resize` on CPU path |
| `-DENABLE_NEON=ON` | ON (aarch64) | ARM NEON SIMD instructions. 2-4x speedup for pixel-level operations (color conversion, thresholding, morphology) on ARM CPUs |

**Verified NEON dispatch (from build output):**
```
Baseline:                    NEON FP16
  required:                  NEON
Dispatched code generation:  NEON_DOTPROD NEON_FP16 NEON_BF16
```
OpenCV auto-detects and compiles optimized kernels for NEON_DOTPROD (dot product acceleration), NEON_FP16 (half-precision), and NEON_BF16 (bfloat16). These are runtime-dispatched based on CPU capability.

### I/O and Media

| Flag | Value | Effect |
|------|-------|--------|
| `-DWITH_GSTREAMER=ON` | ON | GStreamer pipeline support for camera input. Enables hardware-accelerated video decode on Jetson via NVDEC |
| `-DWITH_FFMPEG=ON` | ON | FFmpeg for video file I/O. Hardware decode/encode via NVDEC/NVENC when available |
| `-DWITH_V4L=ON` | ON | Video4Linux2 for direct USB camera access (RealSense, webcams) |
| `-DWITH_OPENGL=ON` | ON | OpenGL interop for GPU texture sharing. Zero-copy display of `cv2.cuda_GpuMat` |
| `-DWITH_GTK=ON` | ON | GTK3 for `cv2.imshow()` window display (requires X11 forwarding in Docker) |

**How to use GStreamer pipeline on Jetson:**
```python
cap = cv2.VideoCapture(
    "v4l2src device=/dev/video0 ! video/x-raw,width=1280,height=720 ! "
    "videoconvert ! video/x-raw,format=BGR ! appsink",
    cv2.CAP_GSTREAMER
)
```

### Features

| Flag | Value | Effect |
|------|-------|--------|
| `-DOPENCV_ENABLE_NONFREE=ON` | ON | SIFT/SURF patented algorithms for feature detection. Better matching quality than ORB for some tasks |
| `-DBUILD_opencv_rgbd=OFF` | OFF | Disabled: uses legacy `glRenderbufferStorageEXT` incompatible with GLVND. Use Open3D for 3D reconstruction instead |

### FP16 Half-Cast Patches

Applied from [jetson-containers](https://github.com/dusty-nv/jetson-containers/blob/master/packages/cv/opencv/build.sh):

```cpp
// normalize_bbox.hpp: (float)weight != 1.0f   (was: weight != 1.0)
// region.hpp:         (float)nms_iou_threshold > 0.0f  (was: nms_iou_threshold > 0)
```

Without these patches, FP16 DNN inference (`DNN_TARGET_CUDA_FP16`) can produce incorrect NMS results or silent data corruption. Always use FP16 on Jetson Thor for throughput -- these patches make it safe.

### OpenCV CUDA Shadow Prevention

The Dockerfile creates a dummy `opencv_python_headless` dist-info after the source build. Without this, `uv pip install` would pull a CPU-only `opencv-python-headless` from PyPI and overwrite the CUDA-enabled `cv2.so`. This is the single most important build correctness measure -- it prevents the entire OpenCV CUDA build from becoming dead code.

## Open3D CUDA Build Flags

### Core CUDA

| Flag | Value | Effect |
|------|-------|--------|
| `-DBUILD_CUDA_MODULE=ON` | ON | CUDA-accelerated tensor operations, TSDF integration, ICP registration, raycasting, nearest neighbor search |
| `-DCMAKE_CUDA_ARCHITECTURES=110` | sm_110 | Targets Jetson Thor specifically |
| `-DCMAKE_CXX_STANDARD=17` | C++17 | Required for CUDA 13.0 compatibility |
| `-DCMAKE_CUDA_STANDARD=17` | CUDA 17 | Same |

**How to use CUDA tensors in code:**
```python
import open3d as o3d
import open3d.core as o3c

pcd = o3d.t.geometry.PointCloud(o3c.Device("CUDA:0"))
pcd.point.positions = o3c.Tensor(points, dtype=o3c.float32, device=o3c.Device("CUDA:0"))

tsdf_volume = o3d.t.geometry.VoxelBlockGrid(
    attr_names=('tsdf', 'weight', 'color'),
    attr_dtypes=(o3c.float32, o3c.float32, o3c.float32),
    attr_channels=((1,), (1,), (3,)),
    voxel_size=0.005,
    block_resolution=16,
    block_count=50000,
    device=o3c.Device("CUDA:0")
)
```

### PyTorch Interop

| Flag | Value | Effect |
|------|-------|--------|
| `-DBUILD_PYTORCH_OPS=ON` | ON | GPU-accelerated point cloud ops as PyTorch extensions. Voxelization, KNN, radius search, sparse convolution, NMS, ball query |
| `-DBUNDLE_OPEN3D_ML=ON` | ON | Open3D-ML model zoo: PointPillars (3D object detection), SparseConvUnet (semantic segmentation), RandLA-Net, PVCNN |
| `-DTORCH_CUDA_ARCH_LIST=11.0` | 11.0 | PyTorch CUDA arch for custom ops compilation |

**How to use PyTorch ops:**
```python
import open3d.ml.torch as ml3d

voxel_size = 0.05
points_tensor = torch.from_numpy(points).cuda()

voxelized = ml3d.ops.voxelize(points_tensor, row_splits, voxel_size)
neighbors = ml3d.ops.knn_search(points_tensor, queries_tensor, k=16)
```

### Sensor and I/O

| Flag | Value | Effect |
|------|-------|--------|
| `-DBUILD_LIBREALSENSE=ON` | ON | Intel RealSense D400/L500 camera integration. Direct RGBD capture, depth alignment, point cloud generation |
| `-DBUILD_GUI=ON` | ON | 3D visualization with Filament rendering engine. Requires full OpenGL (available on Jetson Thor) |
| `-DUSE_BLAS=ON` | ON | OpenBLAS+LAPACK for optimized matrix decomposition (SVD, eigenvalues, least squares) |

**How to use RealSense with Open3D:**
```python
import open3d as o3d

rs_config = {"serial": "", "color_resolution": "1280,720", "depth_resolution": "1280,720"}
rs = o3d.t.io.RealSenseSensor()
rs.init_sensor(rs_config)
rs.start_capture()

frame = rs.capture_frame(wait=True)
rgbd = frame.to(o3d.core.Device("CUDA:0"))
```

### Build Optimisation

| Flag | Value | Effect |
|------|-------|--------|
| `-DBUILD_SHARED_LIBS=ON` | ON | Shared libraries: smaller binary, faster linking, allows `LD_PRELOAD` overrides |
| `-DCMAKE_C_COMPILER=clang` | clang 18 | Required for Filament. Often produces faster ARM64 code than GCC |
| `-DUSE_SYSTEM_CURL=ON` | ON | Uses system curl+OpenSSL instead of bundled BoringSSL. Prevents symbol conflicts |

### Jetson Thor Patches

**FindPytorch.cmake arch conversion**: Open3D's `translate_arch_string` macro converts `CMAKE_CUDA_ARCHITECTURES=110` to `TORCH_CUDA_ARCH_LIST=1.1+PTX0` (incorrect). Patched to read from `TORCH_CUDA_ARCH_LIST` env var directly. This affects any compute capability >= 10.0.

**stdgpu upgrade**: v0.19.0's bundled stdgpu is incompatible with CUDA 13.0 on aarch64. Using `main` branch which includes the fix ([PR #7398](https://github.com/isl-org/Open3D/pull/7398)).

## Build Cache Architecture

### Build Log

Always pipe build output to `/tmp/open3d-build.log`:
```bash
./build.sh 2>&1 | tee /tmp/open3d-build.log
```

With parallel compilation (`-j$(nproc)`), errors from one target get buried under output from other targets running simultaneously. Check the log before running the container:

```bash
# Find actual errors (not warnings)
grep -i "error:" /tmp/open3d-build.log | grep -v warning

# Verify OpenCV CUDA
grep "CUFFT CUBLAS FAST_MATH" /tmp/open3d-build.log

# Verify Open3D
grep "Open3D package found" /tmp/open3d-build.log
```

### Docker Layer Cache

Heavy layers (OpenCV, Open3D) are placed early in the Dockerfile. Changes to `pyproject.toml` or `capx/` code only invalidate later layers. OpenCV and Open3D don't rebuild unless their cmake flags change.

### BuildKit Cache Mounts

```dockerfile
RUN --mount=type=cache,target=/root/.ccache ...   # C++ object cache
RUN --mount=type=cache,target=/root/.cache/uv ...  # Python wheel cache
```

These persist across Docker layer invalidations. If a Dockerfile change forces OpenCV to "rebuild", ccache serves the compiled objects from cache (~5min instead of ~60min).

### Named Volumes (Runtime)

`capx-ccache` and `capx-uv-cache` in docker-compose persist caches for the entrypoint's editable reinstall.

## Runtime Performance Tips

### Use FP16 Inference

All CUDA-built models benefit from FP16 on Jetson Thor (native FP16 tensor cores):

```python
# OpenCV DNN
net.setPreferableTarget(cv2.dnn.DNN_TARGET_CUDA_FP16)

# PyTorch
with torch.cuda.amp.autocast():
    output = model(input_tensor)

# Open3D tensors
tensor = o3c.Tensor(data, dtype=o3c.float16, device=o3c.Device("CUDA:0"))
```

### Keep Data on GPU

Avoid CPU-GPU transfers. Chain operations on `cv2.cuda_GpuMat` or `torch.Tensor.cuda()`:

```python
gpu_frame = cv2.cuda_GpuMat()
gpu_frame.upload(frame)
gpu_resized = cv2.cuda.resize(gpu_frame, (640, 480))
gpu_gray = cv2.cuda.cvtColor(gpu_resized, cv2.COLOR_BGR2GRAY)
# only download the final result
result = gpu_gray.download()
```

### Open3D CUDA Tensors

Use `o3d.t.geometry` (tensor-based) instead of `o3d.geometry` (legacy CPU):

```python
# Slow (CPU, legacy API)
pcd = o3d.geometry.PointCloud()
pcd.points = o3d.utility.Vector3dVector(points)

# Fast (GPU, tensor API)
pcd = o3d.t.geometry.PointCloud(o3c.Device("CUDA:0"))
pcd.point.positions = o3c.Tensor(points, device=o3c.Device("CUDA:0"))
```

### GStreamer Hardware Decode

On Jetson Thor, use GStreamer with NVDEC for zero-CPU-overhead video decode:

```python
pipeline = (
    "filesrc location=video.mp4 ! qtdemux ! h264parse ! nvv4l2decoder ! "
    "nvvidconv ! video/x-raw,format=BGRx ! videoconvert ! "
    "video/x-raw,format=BGR ! appsink"
)
cap = cv2.VideoCapture(pipeline, cv2.CAP_GSTREAMER)
```

### Shared Memory

Docker compose sets `shm_size: 8g`. This is critical for PyTorch DataLoader with `num_workers > 0` and for Open3D's CUDA cached memory manager.

### Profile Switching

Switch server profiles without rebuilding. Only load the servers you need to save GPU VRAM:

```bash
# Testing perception only (SAM3 + GraspNet + PyRoKi) -- ~8GB VRAM
CAPX_PROFILE=default docker compose -f docker-compose.capx.yml up -d

# Full stack (+ OWL-ViT + SAM2) -- ~20GB VRAM
CAPX_PROFILE=full docker compose -f docker-compose.capx.yml up -d
```

## Docker Compose Runtime Optimisations

From `docker-compose.capx.yml`:

| Setting | Value | Effect |
|---------|-------|--------|
| `runtime: nvidia` | nvidia | Exposes GPU to container via NVIDIA Container Toolkit |
| `shm_size: "8g"` | 8GB | Shared memory for PyTorch DataLoader workers and Open3D CUDA memory manager. Default 64MB causes "bus error" with multi-worker data loading |
| `init: true` | true | Runs `tini` as PID 1 for proper signal forwarding and zombie process reaping. Ensures clean shutdown of OpenRouter proxy background process |
| `MUJOCO_GL=egl` | egl | EGL offscreen rendering for MuJoCo/robosuite in headless Docker. No X11 needed for simulation |
| `CAPX_PROFILE` | env var | Switch between `default`/`full`/`minimal` server sets without rebuilding |
| `capx-ccache` volume | named | Persists C++ compilation cache across `docker compose down/up` for entrypoint reinstalls |
| `capx-uv-cache` volume | named | Persists Python package cache across container recreations |
| `healthcheck` | SAM3 :8114 | Probes SAM3 health endpoint every 30s with 120s startup grace (models take time to load) |

## What's NOT Accelerated (and why)

| Component | Acceleration | Reason |
|-----------|-------------|--------|
| PyRoKi IK | CPU (JAX JIT) | `jax<0.4.30` pin has no CUDA 13.0 aarch64 wheels. Upgrade to JAX 0.8+ for GPU |
| opencv_rgbd | Disabled | Legacy OpenGL EXT calls incompatible with GLVND. Use Open3D TSDF/ICP instead |
| Open3D WebRTC | Disabled | Not ported to ARM Linux |
| Intel IPP/ISPC | N/A | Intel x86 only, not available on aarch64 |
| SYCL | N/A | Intel GPU programming model, irrelevant with CUDA |
